### #I-001: 本地环境缺少 pnpm
- **时间**: 2026-03-04T13:27:26+08:00
- **问题描述**: 执行脚手架初始化时发现 `pnpm` 不可用（command not found）。
- **解决方案**: 构建阶段先使用 `npm` 完成依赖安装与脚本执行，保证任务连续推进。
- **影响**: 不影响当前功能实现与测试；后续可在打磨阶段补充 pnpm 锁定与迁移。

### #I-003: ConfigPanel 单测依赖全局 chrome 导致顺序敏感失败
- **时间**: 2026-03-04T14:56:06+08:00
- **问题描述**: 全量测试时 `ConfigPanel` 的“保存成功提示”用例在部分执行顺序下报 `chrome is not defined`。
- **解决方案**: 在 `tests/unit/configPanel.test.tsx` 中显式 mock `chrome.runtime.sendMessage`，消除对其他测试的隐式依赖。
- **影响**: 测试稳定性提升；全量测试可重复通过。

### #I-005: API Base URL 包含 `/v1` 时会重复拼接路径
- **时间**: 2026-03-04T15:03:48+08:00
- **问题描述**: `APIClient` 固定追加 `/v1/chat/completions`，当用户配置 `https://.../v1` 或完整 completions 路径时会生成错误 endpoint。
- **解决方案**:
  - 在 `src/background/engine/api.ts` 中新增 endpoint 归一化逻辑，兼容三种输入：
    - `https://host`
    - `https://host/v1`
    - `https://host/v1/chat/completions`
  - 在 `tests/unit/apiClient.test.ts` 增加回归测试覆盖上述场景。
- **影响**: 提升 OpenAI 兼容服务可配置性，避免请求 404/400 的功能性故障。
